# LifeSync Tracker ‚Äî Code Overview (V6.5)

A walkthrough of what the codebase currently does, file by file and section by section. Reflects the cumulative refactors: **V3** (per-entry data model, channel picker, continuous-flow calendar, warm Honey/Cream palette), **V4** (theme + accent personalisation, Kanban timeline mode, two new Insights charts, date-range filter), **V5** (Stream mode + Insights 3-tab restructure), **V6** (Sleep category + extensible Custom Categories with per-category delete sentinel), **V6.4** (AM/PM time-band whisper gradient on Stream rows + channel-chip hover lift), and **V6.5** (curated Insights cards + Inline Category Builder in Log view + massive-typography summary headlines).

## What the App Is

**LifeSync Tracker** is a single-page, mobile-first web app (installable as a PWA via `manifest.json`) for logging personal **energy levels**, **emotions**, **symptoms**, **activities**, **notes**, **sleep**, and any **user-defined category**. Everything persists to `localStorage`, so it runs fully offline with no backend.

Project files:

| File                     | Purpose                                                                                 |
|--------------------------|-----------------------------------------------------------------------------------------|
| `index.html`             | DOM shell: header, log/save-bar, four views, modal, bottom nav, inline-builder sheet    |
| `styles.css`             | Warm palette variables + safe-area utilities + cat-* tints + tod-* + builder-swatch     |
| `app.js`                 | All behaviour in ~16 numbered sections + tail; ~2700 lines                              |
| `manifest.json`          | PWA manifest (standalone display, iPhone installable)                                   |
| `serve.sh`               | Tiny local dev-server wrapper (autopicks `python3` / `python` / `npx` / `php`)           |
| `README.md`              | Project overview + Run-locally guide + cache-busting tips                               |
| `project_Description.md` | The V6.5 PRD this app is built against                                                  |

Styling is mostly Tailwind CSS loaded from a CDN script tag in `index.html`; `styles.css` provides the small set of CSS variables and utility classes Tailwind doesn't know about (`pt-safe`, `pb-safe`, `cat-*`, `shadow-soft`, `.ios-card`, `.stat-headline`, `.builder-swatch`, `.tod-*`).

## `app.js` ‚Äî The Whole App

`app.js` is structured into ~16 numbered sections (regular) + a tail-end cluster for V6.5 additions. Below is what each section currently does.

### 1. Default Datasets & State

- **`DEFAULT_EMOTIONS`** ‚Äî six starter emotions split by valence (3 positive, 3 negative).
- **`DEFAULT_SYMPTOMS`** ‚Äî four starter somatic symptom labels.
- **`DEFAULT_ACTIVITIES`** ‚Äî four starter behaviours (caffeine, workout, alcohol, meditation).
- **`DEFAULT_EXTRA_CATEGORIES`** (V6) ‚Äî extensible category registry starter. Ships with **Sleep** as the first example: `{ id: 'sleep', name: 'Sleep', type: 'scale', scaleMin: 0, scaleMax: 12, color: '#7C9CB1' }`. On first load, the user's `state.userSettings.customCategories` is seeded with this list.
- **`FALLBACK_LABEL`** ‚Äî human-readable copy for the four sentinel ids (`fallback_positive`, `fallback_negative`, `fallback_symptom`, `fallback_activity` + V6 `fallback_custom`). When a custom descriptor is deleted, every historical reference to it gets rewritten to one of these sentinels so the data still renders meaningfully.
- **`SEVERITY_COLOR` and `SEVERITY_TEXT_COLOR`** ‚Äî warm palette parallels for the three severity buckets.
- **`CAT_CARD_CLASS`** ‚Äî maps category names (`energy`, `pos`, `neg`, `mild`, `mod`, `sev`, `activity`, `note`) to the corresponding `cat-*` classes declared in `styles.css`. New categories get one new class + one entry here.
- **`generateMockData()`** ‚Äî returns ~95 per-entry records scattered across 8 recent days. The first 4 days are hand-shaped to demonstrate cross-channel correlation (morning slump ‚Üí coffee ‚Üí lunchtime recovery ‚Üí afternoon fatigue ‚Üí evening wind-down). The remaining 6 days are generated by an IIFE that uses the day-offset modulo for variety, producing ~12 entries per day with realistic time-of-day distribution (morning burst 7‚Äì9, midday plateau 12‚Äì14, evening wind-down 18‚Äì21, sleep 22‚Äì23).
- **`state`** ‚Äî single in-memory object: `{ userSettings, dailyLogs }`. Each `dailyLog` is a flat event record `{ id, date, type, value, severity? }`.
- **`activeCalendarMode`** ‚Äî tracks `'month' | 'timeline' | 'day' | 'kanban' | 'stream'` in the History view.
- **`insightsRange`** ‚Äî `'all' | '7d' | '30d' | '90d'`, mirrored to `state.userSettings.preferences.insightsRange`.
- **`BUILDER_SWATCHES`** (V6.5) ‚Äî six curated colors for the inline category-builder swatch row (Terracotta, Sage, Dusty Blue, Mocha, Brick, Deep Amber). `currentBuilderColor` is the user's currently-selected hex.

### 2. Initialization (V3 + V4 + V6 migrations)

Runs once on `load`:
1. `resetLogDatePicker()` fills the date input with `now` (timezone-correct).
2. Read `localStorage['lifesync_data']` and parse into `state`.
3. Validate `state.userSettings` + `state.meta`.
4. **First-run seeding** (idempotent via `meta.seeded`): seed emotion/symptom/activity lists if empty; seed `generateMockData()` if zero logs.
5. **V2 ‚Üí V3 migration** (idempotent via `meta.migratedToV3`): explode any V2 *daily card* into one record per channel.
6. **V3 ‚Üí V4 preferences migration** (idempotent via `meta.migratedToV4`): validate theme + accent + insightsRange values.
7. **V6 seed** (idempotent implicit): if `customCategories` is missing, seed it with `DEFAULT_EXTRA_CATEGORIES` (one entry: Sleep).
8. Persist + render.

### 3. Navigation Controller

- **`switchTab(tabId)`** ‚Äî hides all four views, shows the chosen one, recolors the bottom-nav tab, updates the header title, and re-renders the appropriate subview. Save-pill visibility is keyed off the log tab.
- **`setCalendarMode(mode)`** ‚Äî toggles monthly / timeline / day / kanban / stream container and recolors the 5 mode-switcher pills.
- **`refreshCalendarUI()`** ‚Äî dispatches to the correct calendar subview based on `activeCalendarMode`.

### 4. Logger Engine ‚Äî Channel Picker + Per-Channel Forms (V3; V6 extended; V6.5 inline builder)

The Log view splits the V2 single-form into a channel picker plus narrow per-channel panels. Only the panel matching `currentChannel` is visible.

- **`currentChannel`** ‚Äî module-level state, one of the 6 built-ins OR a custom category id.
- **`CHANNEL_LABEL`** ‚Äî maps channel ids to display names.
- **`renderLogger()`** ‚Äî entry point. Re-paints the channel picker chips, all per-channel content lists, and dynamically injects custom-category panels via `ensureCustomCategoryPanels()`.
- **`renderChannelPicker()`** ‚Äî recreates the chips from `getAllCategoryDefs()` (5-7 chips depending on how many custom categories exist). Active chip uses the terracotta accent with `shadow-soft` + `border-[#B47A3C]`; inactive chips are white pills on cream.
- **`renderEmotionsInLogger()` / `renderSymptomsInLogger()` / `renderActivitiesInLogger()`** ‚Äî per-channel content population. Click toggles a colored `bg-` state; severity buttons similarly toggle one of three solid backgrounds.
- **`saveCurrentLog()`** ‚Äî the heart of V3. Reads the *visible panel only* and writes **one discrete entry** to `state.dailyLogs`. For multi-select panels (emotion / activity) it expands into one entry per selected item, all stamped with the same date+time. After save, it briefly flips the Save button to "‚úì Saved" with a sage-green background and resets the channel-specific input.
  - Built-in channels each have an explicit branch (`energy` / `emotion` / `symptom` / `activity` / `note` / `sleep`).
  - Custom channels fall through to the final `else`, which uses `getAllCategoryDefs().find(d => d.key === currentChannel)` and dispatches on `def.valueType` (scale / binary / note).

#### 4b. Inline Category Builder (V6.5)

A major discoverability upgrade. The `+ New custom category` pill below the channel picker expands a fully-styled, in-place form rather than navigating to Settings.

- **`openInlineCategoryBuilder()`** ‚Äî unhides `#inline-builder`, renders the 6 swatches, focuses the name input, updates type-conditional fields.
- **`closeInlineCategoryBuilder()`** ‚Äî hides the sheet and resets name / min / max / type inputs + `currentBuilderColor`.
- **`renderBuilderSwatches()`** ‚Äî paints the 6 swatch buttons with `BUILDER_SWATCHES` data; the active one has `.on` class lifted with a terracotta stroke ring.
- **`pickBuilderColor(hex)`** ‚Äî set `currentBuilderColor`, re-render swatches.
- **`pickBuilderType(type)`** ‚Äî update hidden `inline-builder-type` input, toggle `.on` class on the segmented control buttons.
- **`updateBuilderTypeFields()`** ‚Äî show Min/Max row only when `type === 'scale'`.
- **`saveInlineCategory()`** ‚Äî validate name (non-empty + no duplicate against existing custom categories), push to `state.userSettings.customCategories`, register into `ENTRY_TYPES` (idempotent guard), set `CHANNEL_LABEL[newId]`, seed `streamFilters[newId] = true`, persist, then auto-switch: `currentChannel = cat.id; renderLogger();`. The new chip appears at the end of the picker; the panel flips to the new category's value-type panel (slider / toggle / textarea).

This is the V6 `addCustomCategory()` path with one UX change: kept the user *in flow*.

#### 6c. Stream Mode (V5; V6.4 polish)

**Time is the spine.** 24 hour-rows stack vertically (00:00 at top, 23:59 at bottom). 5+ category columns sit side-by-side horizontally: Energy / Emotion / Symptom / Activity / Sleep + every user-defined custom category, in column order. Each (time-row, category-column) cell contains 0, 1, or a stack of small dots whose `opacity` encodes severity/value (Energy: 0.2 faint ‚Üí 1.0 prominent; Symptom: 0.4 mild ‚Üí 1.0 severe; Emotion: 1.0 positive / 0.55 negative; Activity+Note: 0.5 context). Cluster of 5+ dots in a cell renders a `+N` badge.

Helpers:
- `STREAM_COLS` ‚Äî array of `{ key, label, color }` definitions for the built-in 5 channels. Sleep is appended as the 6th.
- `getCustomCategoryByKey(key)` ‚Äî lookup a user-defined category by its id.
- `getStreamChannels()` ‚Äî runtime merger: STREAM_COLS + state.userSettings.customCategories filtered by id (so a Sleep seed in DEFAULT_EXTRA_CATEGORIES doesn't duplicate the built-in Sleep column).
- `streamOpacity(log)` ‚Äî per-channel opacity encoding (the function used by the dot loop).
- `ensureStreamState()` ‚Äî initializes `state.userSettings.preferences.streamFilters` with `true` for every channel the first time it's called.
- `stepStreamDate(delta)` / `jumpToTodayStream()` ‚Äî date-stepper buttons.
- `toggleStreamFilter(key)` ‚Äî toggle a column on/off via the chips below the day-stepper.
- `todayKeyLocal()` ‚Äî YYYY-MM-DD helper.

**V6.4 whisper gradient** via `todBand(h)`:
- h ‚àà [0,4] ‚Üí `tod-night` (deep navy warm)
- h ‚àà [5,7] ‚Üí `tod-dawn` (peach)
- h ‚àà [8,11] ‚Üí `tod-morning` (gold cream)
- h ‚àà [12,16] ‚Üí `tod-afternoon` (neutral cream)
- h ‚àà [17,20] ‚Üí `tod-dusk` (amber wash)
- h ‚àà [21,23] ‚Üí `tod-night`

Each `.stream-row` receives `todBand(h)` as an extra CSS class. The CSS paints the row with a 0.04‚Äì0.10 alpha linear-gradient (top‚Üíbottom). Net effect: the timeline *visually breathes time-of-day* without screaming.

### 9. Insights Engine (V5: 3-tab; V6.5 curated)

#### 9a. Tab architecture

`INSIGHTS_TAB_RENDERERS` ‚Äî V6.5-curated registry:
```js
const INSIGHTS_TAB_RENDERERS = {
    trends:    ['renderDayOfWeekChart', 'renderRoutineImpact'],
    patterns:  ['renderTopInfluencers', 'renderBehavioralCorrelationGrid', 'renderTimeToEffect', 'renderSymptomCooccurrenceChart']
};
```

The previous `calendar` sub-tab was retired in V6.5 (its card, `renderCalendarSeverityMap`, overlapped with what the Month calendar view already shows).

**`renderInsights()`** ‚Äî clears `#insights-charts`, paints the 2 sub-tab pills (Trends / Patterns), saves the active tab to `preferences.insightsTab`, and dispatches each render function in `INSIGHTS_TAB_RENDERERS[activeTab]` via `window[name]()`.

**`setInsightsTab(tab)`** ‚Äî updates state, persists to `localStorage`, re-renders.

**`applyDateRangeFilter(entries)`** ‚Äî applies the active date range ('all' / 7d / 30d / 90d) and returns the filtered subset.

#### 9b. Emergency summary card (V6.5 simplification)

**`renderInsightSummaryV2()`** ‚Äî replaces the prior 4 small stat cards with TWO large 44 px-black headlines:
- **Average Energy** ‚Äî color-coded: ‚â•7 deep amber, 4‚Äì6 terracotta, <4 brick; na if no energy entries.
- **Your Best Day** ‚Äî auto-detects day-of-week with the highest average energy; requires ‚â•7 days of energy data to surface a name; otherwise shows a "need 7+ days" placeholder.

Wraps each in a `.ios-card.p-5` with a 10 px-black uppercase eyebrow label + the massive number + a small explanatory caption.

#### 9c. Trends tab (V6.5 curation)

**`renderRoutineImpact()` ‚Äî NEW V6.5.** For each activity:
- Bucket `state.dailyLogs` by date into `withDays` (days where the activity was logged) and `withoutDays` (days it wasn't).
- Compute avgEnergy + positive-emotion density per bucket.
- Compute `eDelta = avgEnergy(with) ‚Äì avgEnergy(without)` + `positivity pDelta`.
- Skip rows where neither delta is meaningful.
- Render as a card per activity with two big numbers (Energy delta in /10 + Positivity delta as /day, color-coded green/red/grey).

Earns its keep: surfaces hidden behavioral correlations like "Workout ‚Üí +1.8 energy" the user couldn't see by reading individual rows.

#### 9d. Patterns tab (V6.5 curation)

**`renderTopInfluencers()`** ‚Äî KEPT, enhanced. Computes per-day energy-mood covariation with each descriptor. Renders cards that *say* the delta in plain language (e.g. "Meditation correlates with +2.1 energy boost") with sample-size pills (`5 days vs 23 days`).

**`renderBehavioralCorrelationGrid()`** ‚Äî KEPT, enhanced. 2D grid of (descriptor √ó energy bucket) with a stronger warm-heat gradient so highest-impact cells pop immediately.

**`renderTimeToEffect()` ‚Äî NEW V6.5.** For each (activity, symptom) pair:
- For every activity log, count how many symptom logs of that id subsequently occur within a 120-min window.
- `ratio = clustered / activityCount`.
- Skip pairs with <2 clustered or <20% ratio.
- Surface as cards: "70% of the time, Caffeine leads to Headache within 120 min ‚Äî 7 of 10 Caffeine sessions were followed by Headache."

Earns its keep: temporal causal signal the eye can't see in 30 rows of entries.

**`renderSymptomCooccurrenceChart()`** ‚Äî KEPT. For users tracking complex health cascades, a co-occurrence matrix shows which symptoms often show up together.

#### 9e. [Retired V6.5]

- `renderEnergyTrendChart` (V3) ‚Äî DROPPED; the line on the page was redundant with Stream dots.
- `renderMultiAxisChart` (V4) ‚Äî DROPPED; over-engineered for the use case.
- `renderCalendarSeverityMap` + the `calendar` sub-tab ‚Äî DROPPED; the Month calendar IS the calendar insight.

## `index.html` ‚Äî The Skeleton

The DOM in turn:

1. `<body>` ‚Äî flex column, full viewport height, warm cream background.
2. `<header>` ‚Äî frosted blur bar with `pt-safe`. Contains the dynamic title (`#app-title`) and the hidden Calendar mode-switcher (Month / List / Day / Kanban / Stream).
3. `<main id="main-content">` ‚Äî scroll region; holds the four stacked `<section>` views:
   - `#view-log` ‚Äî date input, **channel picker host (`#channel-picker`)**, **the new `+ New custom category` builder-open button (`#builder-open-btn`)**, **the inline-builder sheet (`#inline-builder`)** with name / swatches / segmented type / conditional min-max / Cancel-Save buttons, then the 6+ `<div data-channel-panel="‚Ä¶">` blocks (only `energy` starts visible).
   - `#view-calendar` ‚Äî `#calendar-month-container` + `#calendar-timeline-container` (which hosts Stream / Day / List / Kanban modes).
   - `#view-insights` ‚Äî range chips (7d/30d/90d/All) ‚Üí 2 sub-tab pills (Trends / Patterns) ‚Üí `#insights-charts` (cards injected here) ‚Üí `#insights-summary` (the 2 huge headline cards).
   - `#view-settings` ‚Äî three sub-sections (Emotions / Symptoms / Activities) + Custom Categories + theme + accent + reset.
4. `#log-save-bar` ‚Äî fixed-position floating pill.
5. `#day-modal` ‚Äî fixed, bottom-aligned sheet.
6. `<nav>` ‚Äî bottom tab bar with 4 SVG-icon buttons.

`app.js` is loaded with `defer` so the DOM is fully parsed before it runs.
